02 — Component Breakdown
What each service is responsible for
📄
GitHub Pages
Static File Hosting

Hosts a single index.html file — nothing else. GitHub Pages is a free static web server that serves this file to anyone who visits your URL.

It has zero backend logic. It cannot run code, call APIs, or store data. It simply hands the HTML file to the user's browser and its job is done.

The browser then runs all the JavaScript inside that file, which is what builds the dashboard and calls the Cloudflare Worker for data.

Why GitHub Pages? Free, zero config, deploys on every git push, HTTPS included, custom domain support available.

⚙️
Cloudflare Worker
Backend Logic & Orchestration

The entire backend — all logic lives here. It runs on Cloudflare's global edge network (300+ data centres), executing in milliseconds near the user.

It receives a request like GET /api/sentiment?game=codm, checks the cache, fetches Reddit, calls OpenAI, builds the response, and returns JSON.

It also protects your OpenAI key — the key never touches the browser. The Worker holds it as an encrypted environment secret.

Why a Worker and not just calling APIs from the browser? CORS restrictions, API key security, and server-side caching all require a backend layer.

🔑
OpenAI API Key
Authentication Credential

The key is a bearer token — it tells OpenAI's servers "this request is authorised to use my account and bill me." Without it, all API calls return 401 Unauthorized.

It is stored as a Cloudflare Worker Secret — encrypted at rest, injected as an environment variable at runtime, never visible in logs or code.

The key itself doesn't do any AI work. It's purely authentication. The actual intelligence is the model (gpt-4o-mini) running on OpenAI's servers.

Cost: gpt-4o-mini costs ~$0.15 per million input tokens. A full 60-post analysis uses roughly 2,000 tokens = fractions of a cent per run.

📋
Reddit JSON API
Data Source

Reddit exposes a free, public, unauthenticated JSON API. Any subreddit URL can have .json appended to return machine-readable post data.

The Worker fetches 4 endpoints per game — hot and new for each of the two subreddits — all in parallel using Promise.all().

Posts are then deduplicated by post ID (since hot and new often overlap) and stickied mod posts are stripped out before sending to OpenAI.

No Reddit account or API key required. The Worker identifies itself with a User-Agent header as good practice.

🧠
Sentiment Analysis
OpenAI gpt-4o-mini

Uses zero-shot prompting — no training data, no fine-tuning. The model is given a system prompt describing the task and a numbered list of post titles.

response_format: json_object forces the model to return only valid JSON — no markdown, no explanation — making it safe to parse directly.

The model returns a score per post (−1.0 to +1.0), a category, and a summary object with percentages, top topics, a mood sentence, and a trending issue.

Why gpt-4o-mini? It understands gaming slang, community context, and nuanced frustration (e.g. "this is peak design" is positive) far better than keyword-matching.

💬
Word Cloud
Server + Client Pipeline

Built in two stages: frequency counting on the Worker (server-side), and HTML rendering in the browser (client-side). No third-party library.

The Worker strips stop words, tokenises all post titles, and returns the top 60 words as [word, count] pairs, sorted by frequency descending.

The browser renders each word as a styled HTML span inside a flex-wrap container. Font size scales linearly from 11px to 52px based on frequency ratio.

Why not a canvas library? Libraries like wordcloud2.js have silent failures on dynamic canvas sizing and font loading. Pure HTML is deterministic and always works.

03 — Request Lifecycle
Step-by-step: what happens when you click a game tab
01
User Opens Dashboard / Clicks a Tab
The browser loads index.html from GitHub Pages. This is a single static file — HTML, CSS, and JavaScript all in one. GitHub Pages serves it over HTTPS with no server-side processing whatsoever. The JavaScript runs immediately in the browser and calls fetchData('codm', true).
GITHUB PAGES BROWSER JS
02
Browser Calls the Cloudflare Worker
The browser makes a GET request to https://game_sentinel_multi.YOUR.workers.dev/api/sentiment?game=codm. This hits Cloudflare's edge network — the request is routed to the nearest of Cloudflare's 300+ global data centres, so latency is minimal regardless of where the user is.
CORS HEADERS CLOUDFLARE EDGE
03
Worker Checks the Cache
The Worker looks up a per-game cache key (e.g. game-sentinel-codm-v3) in caches.default — Cloudflare's built-in edge cache. If a result exists and is less than 15 minutes old, it returns immediately with "cached": true in the JSON, skipping all Reddit and OpenAI calls entirely. This is what makes tab-switching fast after first load.
15-MIN TTL PER-GAME KEY
04
Worker Fetches Reddit in Parallel
On a cache miss, the Worker fires 4 fetch calls simultaneously using Promise.all() — hot and new feeds for each of the game's two subreddits. For CODM: r/CODM/hot.json, r/CODM/new.json, r/CallofDutyMobile/hot.json, r/CallofDutyMobile/new.json. Results arrive in ~300–700ms. Posts are deduplicated by ID and stickied posts are filtered out.
PROMISE.ALL DEDUPLICATION REDDIT PUBLIC API
05
Worker Calls OpenAI for Sentiment Scoring
The first 60 deduplicated post titles are formatted as a numbered list and sent to gpt-4o-mini with a system prompt describing the scoring rules. The response_format: json_object parameter guarantees the response is parseable JSON. The model returns a score and category for each post plus the full community summary object. This step takes 5–15 seconds — the dominant cost in the pipeline.
GPT-4O-MINI JSON_OBJECT FORMAT ZERO-SHOT
06
Worker Builds Word Frequencies
Simultaneously with OpenAI (actually right after), the Worker tokenises all post titles — lowercasing, stripping punctuation, splitting on whitespace — then filters against a hardcoded stop-word list (~60 common words like "the", "a", "is"). The remaining words are counted and the top 60 by frequency are returned as [word, count] pairs. This runs in microseconds in pure JavaScript.
SERVER-SIDE STOP-WORD FILTER TOP 60 WORDS
07
Worker Caches and Returns JSON
The complete payload — summary, per-subreddit breakdown, word cloud, and enriched post list — is stored in caches.default with a Cache-Control: max-age=900 header. The same payload is returned to the browser. Any subsequent request within 15 minutes returns from cache without touching Reddit or OpenAI.
CACHE-CONTROL MAX-AGE=900
08
Browser Renders the Dashboard
The browser parses the JSON and calls renderDashboard(data, game). The game name is passed explicitly so the correct colour theme palette is applied. The canvas gauge is drawn, Chart.js renders the doughnut, word cloud spans are injected, topic chips animate in with staggered delays, and the post feed is built. The ticker duplicates its HTML for seamless CSS loop animation. Total client-side render time is under 50ms.

05 — Word Cloud Generation Pipeline
Server-side counting → client-side HTML rendering
1
Input — All Post Titles
Every deduplicated post title from all subreddits is fed into the function. For CODM this is typically 80–120 titles after deduplication. The full text is used, not just the first 60 sent to OpenAI.
2
Normalise & Tokenise (Worker)
Each title is lowercased, then all non-alphanumeric characters are replaced with spaces using .replace(/[^a-z0-9\s]/g, " "). The string is then split on whitespace into individual word tokens. This handles punctuation, special characters, and hyphens uniformly.
3
Stop Word Filter (Worker)
Tokens shorter than 3 characters are dropped. Tokens matching a hardcoded stop-word Set of ~60 common English words (a, the, and, is, it, you, they, very, just, etc.) are removed. A Set is used for O(1) lookup speed. Words like "bug", "patch", "ranked", "meta" survive.
4
Frequency Count & Sort (Worker)
Surviving words are counted into a plain JavaScript object: freq[word] = (freq[word] || 0) + 1. The object is then converted to entries, sorted descending by count, and sliced to the top 60 entries. These are returned in the API response as [["ranked", 14], ["update", 11], ...].
5
Font Size Scaling (Browser)
Each word's font size is calculated as: fontSize = 11 + (count / maxFreq) * 41 — giving a range of 11px to 52px. The most frequent word always renders at maximum size. Opacity scales similarly from 0.55 to 1.0 so less common words visually recede.
6
Deterministic Colour Assignment (Browser)
Each word gets a colour from the game's palette using: palette[charSum % palette.length] where charSum is the sum of all character codes in the word. This means the same word always gets the same colour across re-renders, so the cloud doesn't flicker on refresh.
7
Rotation & Glow Effects (Browser)
Words with ratio < 0.6 and length < 10 characters get a slight rotation: ((charSum * 7 + length * 13) % 31) - 15 degrees — between −15° and +15°. This is also deterministic. A text-shadow glow is applied proportional to frequency. On hover, words snap to 0° rotation and scale to 1.12× via CSS transition.
8
HTML Flex Layout (Browser)
All word spans are appended to a div with display: flex; flex-wrap: wrap; justify-content: center. The browser's layout engine handles placement naturally — no canvas, no collision detection, no library. Larger words push smaller words into available space organically.
06 — Caching Strategy
Three layers of caching keep the dashboard fast
Cache Layer Where Duration Scope Purpose
L1 Cloudflare Edge Cache Cloudflare Worker — caches.default 15 minutes Per game key Avoids calling Reddit + OpenAI on every request. Serves thousands of users from one cached response.
L2 Browser In-Memory Cache JavaScript — gameCache{} object Session only Per game, per tab Switching back to a game you already loaded renders instantly from memory, zero network request.
L3 Auto-Refresh Timer Browser — setTimeout() 15 min interval Current game only Busts both L1 and L2 caches on the active game after 15 minutes, fetching fresh data automatically.
